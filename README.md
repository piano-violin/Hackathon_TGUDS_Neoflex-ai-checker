## О проекте

AI-агент для автоматизированной проверки тестовых заданий кандидатов при поступлении в Учебный центр. Решение позволяет автоматизировать рутинный процесс оценки технических заданий, формировать структурированную обратную связь и выгружать результаты в удобном формате.

### Ключевые возможности

| Функция | Описание |
|---------|----------|
| **Автоматическая проверка** | Обработка ответов кандидатов из файлов различных форматов |
| **Оценка по 10-балльной шкале** | Обоснованная и воспроизводимая система оценивания |
| **Двойной фидбек** | Отдельная обратная связь для кандидатов и для команды Учебного центра |
| **Telegram-бот** | Интерактивное взаимодействие с кандидатами и администраторами |
| **Выгрузка результатов** | Экспорт в Excel для интеграции в бизнес-процесс |
| **Мультиформатный ввод** | Поддержка TXT, PDF, DOCX, XLSX, изображений и других форматов |

## Установка и запуск

### Системные требования

- **Python**: 3.10+
- **PostgreSQL**: 15+
- **RAM**: минимум 16 ГБ (для работы LLM)
- **GPU**: NVIDIA с CUDA 12.6 (рекомендуется для ускорения инференса)

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd <project-folder>
```

### 2. Создание виртуального окружения

```bash
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
# или
.venv\Scripts\activate     # Windows
```

### 3. Установка Python-зависимостей

```bash
pip install -r requirements.txt
```

### 4. Установка системных зависимостей

```bash
# Для конвертации PDF
sudo apt-get install poppler-utils

# Для обработки DOC через LibreOffice
sudo apt-get install libreoffice

# Для OCR (Tesseract)
sudo apt-get install tesseract-ocr
sudo apt-get install tesseract-ocr-rus
```

### 5. Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```env
# Подключение к PostgreSQL
user=your_username
password=your_password
host=localhost
port=5432
dbname=neoflex_testing

# Модель (опционально)
QWEN_MODEL_NAME=Qwen/Qwen2.5-Coder-7B

# Telegram Bot (для bot.py)
BOT_TOKEN=your_telegram_bot_token
ADMIN_PASSWORD=neoflex2026
```

### 6. Подготовка входных данных

Поместите файлы с ответами кандидатов в папку `in/`:

```bash
mkdir -p in
# Скопируйте файлы кандидатов в папку in/
```

Поддерживаемые форматы: `.txt`, `.md`, `.sql`, `.pdf`, `.docx`, `.doc`, `.xlsx`, `.png`, `.jpg`, `.jpeg`

### 7. Запуск пайплайна

```bash
python main.py
```

При запуске программа запросит:
- ФИО кандидата
- Email кандидата

После завершения работы будут созданы:
- Записи в базе данных
- Excel-файл с результатами: `school_results_session_<id>.xlsx`

## Модули системы

### core.py — Ядро бизнес-логики

Единый модуль с функциями, используемыми как в консольном пайплайне, так и в Telegram-боте:

- `register_candidate()` — регистрация кандидата и создание сессии
- `parse_candidate_file()` — парсинг файла с ответами
- `parse_text_answers()` — парсинг текстовых ответов (для бота)
- `save_answers()` — сохранение ответов в БД
- `evaluate_session()` — оценка сессии
- `get_candidate_result()` — получение результата для кандидата
- `get_school_result()` — получение результата для Учебного центра
- `export_results_to_excel()` — экспорт в Excel

### bot.py — Telegram-бот

Модуль для взаимодействия с пользователями через Telegram:

- **Регистрация кандидатов** с валидацией данных
- **Прием ответов** в текстовом формате и файлами
- **Отправка результатов** кандидату после проверки
- **Панель администратора** для просмотра и выгрузки результатов

### model.py — Загрузка LLM

Модуль инициализирует модель **Qwen2.5-Coder-7B** с 4-bit квантованием для эффективного использования памяти:

- Автоматическое определение GPU/CPU
- Кэширование модели в локальную директорию
- Оптимизация параметров генерации

```python
from model import qwen_model, qwen_tokenizer

# Модель и токенизатор доступны глобально
if qwen_model is not None:
    print("Модель успешно загружена")
```

### evaluations.py — Оценка ответов

Основной модуль для оценки ответов кандидатов:

- **LLM-оценка**: Формирование структурированного промпта и парсинг JSON-ответа
- **Rule-based fallback**: Автоматическая оценка по ключевым признакам при ошибках модели
- **Mock-оценка**: Тестовый режим без LLM для отладки пайплайна

```python
from evaluations import evaluate_session_answers

evaluate_session_answers(session_id=1)
```

### prompts.py — Система промптов

Содержит три типа промптов:

| Функция | Назначение |
|---------|------------|
| `evaluate_answer_prompt()` | Оценка ответа на конкретное задание |
| `generate_candidate_final_feedback_prompt()` | Фидбек для кандидата |
| `generate_school_feedback_prompt()` | Отчёт для Учебного центра |

### db.py — База данных

Модуль для работы с PostgreSQL:

- Создание таблиц и схемы БД
- CRUD-операции для кандидатов, сессий, ответов
- Экспорт результатов в Excel

### file_to_json_converter.py — Конвертер файлов

Универсальный модуль для преобразования файлов различных форматов в структурированный JSON:

- **Текстовые файлы**: TXT, MD, SQL
- **Документы**: PDF, DOCX, DOC (через LibreOffice)
- **Таблицы**: XLSX (openpyxl)
- **Изображения**: PNG, JPG, JPEG (OCR через Tesseract)

```python

result = convert_to_json("in/candidate_answers.pdf")
# Возвращает: {"answers": {"1": "...", "2": "...", "3": "...", "4": "..."}}
```

## Тестовые задания

Система проверяет 4 задания для позиции Data Engineer:

| № | Название | Навыки |
|---|----------|--------|
| 1 | Анализ таблицы клиентов | Проектирование БД, нормализация, ключи |
| 2 | Архитектура таблицы товаров | Историческое хранение, SCD2 |
| 3 | Исправление SQL-запроса | SQL, JOIN, агрегации, HAVING |
| 4 | Логическая задача | Нестандартное мышление |

Каждое задание оценивается максимум в 25 баллов, итоговая оценка нормируется к 10-балльной шкале.

## Результаты работы

### Выходные данные

1. **Фидбек для кандидата** — дружелюбный, развивающий текст с:
   - Итоговым баллом
   - Общей оценкой уровня
   - Сильными сторонами
   - Зонами роста и рекомендациями

2. **Отчёт для Учебного центра** — технически подробный отчёт с:
   - Определением уровня (Junior/Middle/Senior)
   - Оценкой архитектурного мышления
   - Анализом работы с SQL
   - Типовыми ошибками
   - Рекомендацией по зачислению

3. **Excel-выгрузка** — таблица со всеми результатами для интеграции в бизнес-процесс.

## Безопасность и этика

- **Корректный тон**: Обратная связь формулируется уважительно и конструктивно
- **Отсутствие токсичности**: Промпты настроены на профессиональный стиль
- **Защита данных**: Персональные данные кандидатов хранятся в защищенной БД
- **Воспроизводимость**: Каждая оценка обоснована и может быть проверена
